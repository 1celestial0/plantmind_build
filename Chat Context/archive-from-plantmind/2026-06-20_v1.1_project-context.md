## 2. PROJECT: PLANTMIND

### What it is
An **agentic Engineering Intelligence layer** that converts industrial sensor streams into ranked, explainable corrective actions — and *proves* each decision by flipping a failing asset from RED to GREEN using counterfactual simulation.

**Engine name:** Götze Decision Engine (GDE)
**Tagline:** *Predict the failure. Decide the fix. Prove it.*
**Dataset:** NASA C-MAPSS (21 sensors, run-to-failure, turbofan degradation)
**Stack:** Python · scikit-learn · Streamlit · Plotly · Claude API · Databricks CE

### 5-Layer Architecture
```
LAYER 1 · DATA         → NASA C-MAPSS → clean → RUL label → 30-cycle sliding window
LAYER 2 · FEATURES     → Rolling mean + std per degrading sensor
LAYER 3 · PREDICTION   → RandomForest → predicted RUL → RED flag if RUL < 30
LAYER 4 · DECISION     → Götze Score → winner action → counterfactual proof ← CORE IP
LAYER 5 · PROVE+LEARN  → RED→GREEN chart → dashboard → weight recalibration
```

### The Core IP: Götze Score Formula
```
G = w₁·ΔHealth + w₂·(1−NormCost) + w₃·(1−NormTime) + w₄·Safety
Weights: health=0.40, cost=0.25, time=0.20, safety=0.15 (must sum to 1.0)
DESIGN RULE: AI does uncertain work. Deterministic math makes the final decision.
```

### Rubric & Score
- Weighted composite: **0.83 / 1.0** → Target: **0.91**
- Top gaps: live demo stability (−0.15), agent trace in UI (−0.10), scalability slide (−0.13)

### Patent 4 Summary (new this session)
5-component architecture on top of MetaGPT:
1. **Research Fetcher Agent (RFA):** Queries GitHub, arXiv, CVE databases, ISA/IEC/NIST standards live
2. **Industrial Knowledge Graph (IKG):** Versioned, typed graph accumulating validated knowledge across projects
3. **Adversarial Validation Agent (AVA):** Checks every agent's output against IKG → Deviation Report (CRITICAL/WARN/INFO)
4. **Suggestion Engine (SE):** "Change X to Y because Standard Z.Section requires it"
5. **Feedback Loop:** Accepted corrections written back into IKG → cross-project institutional learning

**Prior art gap:** No existing tool combines live research + adversarial validation + corrective suggestions + cross-project accumulation.
