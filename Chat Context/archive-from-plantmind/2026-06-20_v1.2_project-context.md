## 2. PROJECT: PLANTMIND

### What it is
An **agentic Engineering Intelligence layer** that converts industrial sensor streams into ranked, explainable corrective actions — and *proves* each decision by flipping a failing asset from RED to GREEN using counterfactual simulation.

**Engine name:** Götze Decision Engine (GDE)
**Tagline:** *Predict the failure. Decide the fix. Prove it.*
**Dataset:** NASA C-MAPSS FD001 (100 turbofan engines, 21 sensors, run-to-failure, RUL clipped at 130)
**Stack:** Python · scikit-learn · Streamlit · Plotly · Claude API · Databricks CE

### 5-Layer Architecture
```
LAYER 1 · DATA         → NASA C-MAPSS → clean → RUL label → 30-cycle sliding window
LAYER 2 · FEATURES     → Rolling mean + std per degrading sensor (13 sensors, window=30)
LAYER 3 · PREDICTION   → RandomForest (n=200, depth=15) → RUL → RED if RUL < 30
LAYER 4 · DECISION     → Götze Engine: score 4 actions → winner → counterfactual proof
LAYER 5 · PROOF+LEARN  → RED→GREEN chart + JSONL decision log + fleet summary
```

### Götze Score Formula (DESIGN RULE: no LLM, deterministic math only)
```
G = 0.40·ΔHealth + 0.25·(1−NormCost) + 0.20·(1−NormTime) + 0.15·Safety
  where: ΔHealth = rul_gain / 130,  NormCost = cost / max_cost,  NormTime = time / max_time
  Safety ∈ [0,1] from action catalog.  Weights sum to 1.0.
```

### SURROGATE_GAIN_MAP (RUL cycles recovered per action)
```python
{"replace_bearing": 95.0, "reduce_load": 40.0, "flush_lubrication": 25.0, "monitor_only": 5.0}
```

## 4. METAGPT LEVEL 2 ARCHITECTURE

### Message spine (`src/messages.py` — 5 frozen dataclasses)
```
SensorReading → EngineFeatures → EngineHealth → MaintenanceAction → EngineDecision
```
All `@dataclass(frozen=True)`. Immutable, hashable, auditable.

### Role boundaries
| Role | Consumes | Produces |
|---|---|---|
| `DataEngineerRole` | raw C-MAPSS path | `SensorReading`, history DataFrame |
| `MLEngineerRole` | `SensorReading` + history | `EngineHealth` |
| `GotzeEngine` | `EngineHealth` (pipeline mode) or `(int, float)` (legacy) | `messages.EngineDecision` or internal `EngineDecision` |
| `ProofEngineerRole` | `messages.EngineDecision` | Plotly chart data, JSONL log, fleet summary |
| `PipelineOrchestrator` | `engine_id: int` | `{decision, proof_chart_data, agent_trace}` |

### Dual-mode GötzeEngine pattern
```python
# Legacy (CLI/tests) — internal EngineDecision
engine.diagnose(engine_id=7, current_rul=18.0)

# Pipeline mode — messages.EngineDecision
engine.diagnose(EngineHealth(...))  # isinstance check dispatches automatically
```
