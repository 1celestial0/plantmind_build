# Counterfactual Proof

## What It Is

A counterfactual proof answers: *"What WOULD HAVE happened if we had taken action X?"*

In PlantMind, it compares two trajectories:
- **Actual:** the engine keeps degrading → hits RUL=0 → failure
- **Counterfactual:** the winning action is applied → RUL recovers → GREEN

The proof is visual: a two-line chart. One line goes down (failure). One line goes up (rescue). This is the **"money shot"** of the demo.

## The Mechanism

```python
# Before action: engine at RUL = current_rul (e.g., 18 cycles)
# After action: engine RUL = current_rul + winner.rul_gain

new_rul = current_rul + winner.rul_gain
is_rescued = new_rul > RED_THRESHOLD  # > 30 cycles = GREEN

# The chart shows:
# - Red line: degradation curve from RUL=18 → 0 (failure)  
# - Green line: degradation curve from RUL=18+40 → 58 (safe zone)
```

## Why This is the Core IP

**Prior art:**
- LIME/SHAP: explain *model predictions* (why the model said RUL=18)
- Prescriptive maintenance papers: suggest actions, no proof
- Digital twins: simulate the asset's future, but don't rank or prove actions

**Our gap:** We generate counterfactual proof *of action efficacy* — not of model predictions. This is a different claim space. When your fix is right, you prove it with math, not assertion.

## Connection to [[Surrogate Twin]]

The counterfactual trajectory is NOT real historical data. It's generated by the [[Surrogate Twin]] — a model trained on historical failure curves that simulates "what would the degradation curve look like if this action were applied at this cycle?"

## WHAT / WHY / HOW / WHEN / WHY NOT

**WHAT:** A visual + numeric proof showing that the winning action rescues the failing engine.

**WHY:** "Trust me, this fix works" is not good enough for a plant shutdown decision worth $100K/hour. The engineer needs to show their boss a chart, not a probability.

**HOW:** `EngineDecision.is_rescued` = `(current_rul + winner.rul_gain) > 30`. The chart plots two exponential decay curves starting from the same point.

**WHEN:** Generated for every RED engine after [[Götze Score]] picks the winner.

**WHY NOT:**
- Show only the score: "Score = 0.89" tells nothing about HOW MUCH better it is.
- Show historical data: we don't have post-repair historical data in C-MAPSS. Counterfactual (surrogate twin) is the only option.
